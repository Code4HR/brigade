Experiment

Experiments are explorations into a civic problem. They involve lots of research, lots of listening to local experts, and lots of throw-away code. Almost every project should start out as an experiment and most projects should stay as experiments.

    Experiment Stage Theme:

  • Make many things!!1!

    Your project is an experiment if:

  • You haven't talked to users yet
  • Your Brigade hasn't endorsed it yet

    Requirements

  • All Brigade projects should be respectful and be cool under the code of conduct.

    What the project team should be doing:

  • Listening to the community about civic problems
  • Researching those problems
  • Lots of experiments

    What the Brigade should be doing:

  • Organizing hack nights
  • Inviting community and government experts in
  • Make infrastructure and resources easily accessible
  • Show off the other experiments

    What Code for America should be doing:

  • Getting donated resources
  • Show off experiments

Alpha

Alpha projects have shown that with a little bit of work, they may be able to actually solve one persons problem. We've also got some recommendations that we've seen as helpful for volunteer built projects.

    Alpha Stage Theme:

  • Solve one person's problem

    Your project is an Alpha if:

  • Your Brigade has approved the project
  • You've got a project owner
  • You've got something online to share

    Requirements

  • A project owner
  • The project is listed on the Code for America API

    What the project team should be doing:

  • Choose a project owner
  • Write a clear README
  • Make the project accessible for others who want to help
  • Start with agile development
  • Start user testing

    What the Brigade should be doing:

  • Approve the project
  • Add to their project list
  • Help with user testing

    What Code for America should be doing:

  • Promote with the Project Search page
  • Get help for the project with the Civic Issue Finder
  • Produce examples and guides for how to write good issues
  • Produce agile and project management content
  • User testing resources and support
  • Project planning office hours

Beta

Beta projects blah blah blah

    Beta Stage Theme:

  • Moaaar users

    Your project is an beta if:

  • You've done user testing
  • You've got a partner

    Requirements

  • A government or community partner

    What the project team should be doing:

  • Talking to users
  • Agile development
  • Get more users
  • Writing tests

    What the Brigade should be doing:

  • Formalize government or community partner
  • Help decide on metrics to measure for impact

    What Code for America should be doing:

  • Engineering office hours

Official

Official projects blah blabb bblah

    Experiment Stage Theme:

  • Impact

    Your project is official if:

  • You can prove the impact

    Requirements

  • A launch party
  • Published analytics and metrics

    What the project team should be doing:

  • Talking to users
  • Polish for launch
  • Analytics

    What the Brigade should be doing:

  • Launch party
  • Help spread project
  • Metrics to measure impact

    What Code for America should be doing:

  • Celebrate launched projects
  • Marketing office hours
  • Analytics support
  • Metrics office hours
